## 1. 인증 (Auth)

### 1.1 로그인 전 (비인증 상태)

```
랜딩 페이지 (/landing)
├── "Google로 로그인" 버튼
│   └── → GET /api/auth/google → Google OAuth → 콜백 → 쿠키 발급 → 앱 진입
└── "이메일로 로그인" 버튼
    └── → 로그인 페이지 (/auth/login)
        ├── 이메일 + 비밀번호 입력 → POST /api/auth/login → 쿠키 발급
        └── "계정 없으신가요?" → 회원가입 페이지 (/auth/register)
            └── 이름 + 이메일 + 비밀번호 → POST /api/auth/register → 쿠키 발급
```

### 1.2 로그인 후 (인증 완료)

```
세션 쿠키 발급 완료
└── auth redirect 복원 (이전 페이지가 있으면 복귀, 없으면 / 로 이동)
    └── 회사 컨텍스트 확인
        ├── 소속 회사 있음 → 대시보드 진입
        └── 소속 회사 없음 → 회사 생성 또는 초대 링크 수락 안내
```

### 1.3 초대 플로우

```
초대 생성 (Settings → 팀 관리)
├── 역할 선택 (admin / member)
├── "초대장 생성" 버튼 → 토큰 생성
└── 초대 링크 복사: /invite/{token}

초대 수락 (초대 링크 접속)
├── /invite/{token} → 초대 정보 프리뷰
│   ├── 회사명, 역할, 만료일 표시
│   └── "초대 수락" 버튼
│       ├── 미로그인 상태 → /auth/login?redirect=... 으로 이동
│       └── 로그인 상태 → 회사 합류 → 워크스페이스 전환
│
└── /invite/enter (코드 직접 입력)
    ├── 초대 링크 또는 토큰 붙여넣기
    └── 미리보기 → 수락
```

    └── 각 항목: 일정명 + 고객사 + 날짜 → 클릭 시 일정 목록
```

## 3. 영업일지 (/sale-list)

### 3.1 목록

```
영업일지 목록
├── 검색 바 (고객사명, 담당자명)
├── 필터 탭
│   ├── 전체
│   ├── 미분석 (AI 분석 전)
│   └── 분석완료
├── 무한 스크롤 목록
│   └── 각 항목: 고객사 + AI 요약 + 방문일 + 분석 상태 배지
└── FAB → "일지 작성" → /sale-list/regi
```

### 3.2 작성 (/sale-list/regi) — 핵심 플로우

```
【입력】
├── VoiceRecorder
│   ├── 음성 녹음 버튼 → 마이크 권한 → 실시간 녹음 → 중지
│   ├── 파일 첨부 버튼 → 오디오 파일 선택 (audio/*, video/webm)
│   └── 업로드 플로우:
│       ├── prepareUpload → presigned PUT URL 발급
│       ├── PUT R2 (브라우저 → R2 직접 업로드)
│       ├── confirmUpload → DB에 파일 메타데이터 저장
│       └── transcribeFile → Groq Whisper STT → 텍스트 반환
│           └── 텍스트가 orig_memo에 자동 삽입
│
├── 텍스트 입력 필드
│   ├── 고객사명 (자동완성 + 유사 고객사 제안)
│   ├── 담당자명
│   ├── 방문 위치
│   ├── 방문 일시
│   └── 내용 (orig_memo)
│
└── 저장 버튼
    ├── "저장" → 일지만 저장
    └── "AI 저장" → 저장 + AI 분석 동시 실행

【저장 시 고객사 매칭】
고객사명 입력했지만 clie_idno 미확정인 경우
├── findBestMatch → 유사 고객사 검색
├── 매칭 결과 있음 → 프리세이브 다이얼로그
│   ├── "맞아요" → 해당 고객사로 연결 후 저장
│   └── "아니에요" → 새 이름 그대로 저장
└── 매칭 없음 → 바로 저장

【AI 분석 결과】
├── 저장 성공 → analyze API 호출
├── LLM이 추출:
│   ├── ai_client_name (고객사명 추출)
│   ├── matched_client_idno (기존 고객사 매칭)
│   ├── appointments (일정 목록 + 자동 등록)
│   ├── pricing (금액 정보)
│   └── summary (요약)
│
└── 분석 후 고객사 연결 다이얼로그
    ├── AI가 고객사 추출했고 아직 미연결인 경우에만 표시
    ├── 매칭 고객사 있음: "'{고객사명}'에 연결할까요?"
    │   ├── "맞아요" → 해당 고객사 연결
    │   └── "아니에요" → 새 고객사 생성 후 연결
    └── 매칭 없음: "'{AI추출명}'을 새 고객사로 등록할까요?"
        ├── "등록" → 신규 고객사 생성 후 연결
        └── "스킵" → 연결 없이 상세 이동
```

### 3.3 상세 (/sale-list/:id)

```
영업일지 상세
├── 기본 정보 (고객사, 담당자, 방문일, 위치)
├── 내용 원문
├── AI 분석 결과 (분석 완료 시)
│   ├── 요약
│   ├── 금액 정보
│   ├── 자동 생성된 일정 목록 (상태 표시)
│   └── 메모/노트
├── 액션 버튼
│   ├── "AI 분석" (미분석 시) → analyze → 결과 표시
│   ├── "수정" → 인라인 편집 모드
│   └── "삭제" → 확인 다이얼로그 → 삭제 후 목록
└── 첨부파일 (음성 파일 등)
```

## 4. 일정 (/schedule)

### 4.1 목록

```
일정 목록
├── 상태별 탭 (전체 / 예정 / 임박 / 지연 / 완료 / 취소)
│   └── 각 탭에 카운트 표시
├── 경고 배너 (지연/임박 일정이 있으면 자동 표시)
├── 무한 스크롤 목록
│   └── 각 항목: 일정명 + 고객사 + 날짜 + 상태 + 액션 버튼
│       ├── "완료 처리" 버튼
│       ├── "취소" 버튼
│       └── "수주 생성" 버튼 (빠른 수주 연결)
└── FAB → "일정 추가" → 등록 다이얼로그
```

### 4.2 CRUD

```
일정 등록/수정 다이얼로그
├── 고객사명 (자동완성)
├── 일정명 (필수)
├── 설명
├── 일시 (필수)
└── 저장 / 수정

일정 상태 변경
├── scheduled → "완료" 클릭 → 확인 → completed
├── scheduled → "취소" 클릭 → canceled
└── overdue (지연) → 완료/취소 처리 가능

일정에서 빠른 수주 생성
├── "수주 생성" 버튼 클릭
├── 수주 빠른 등록 다이얼로그 (금액/품목 입력)
└── 생성 완료 시 해당 일정도 자동 완료 처리
```

## 5. 수주 (/order)

### 5.1 목록

```
수주 목록
├── 상태별 탭 (전체 / 제안 / 협상 / 확정 / 취소)
├── 재무 요약 (파이프라인 총액, 확정 금액)
├── 무한 스크롤 목록
│   └── 각 항목: 고객사 + 품목 + 금액 + 상태 + 액션
│       ├── 상태 변경 버튼 (다음 단계로 이동)
│       └── "납품 등록" 버튼 (confirmed 상태 시 활성화)
└── FAB → "수주 등록" → 등록 다이얼로그
```

### 5.2 CRUD

```
수주 등록/수정 다이얼로그
├── 고객사명 (자동완성, 필수)
├── 품목/서비스명 (필수)
├── 금액 (필수)
├── 상태 선택
├── 계약일
├── 납품 예정일
└── 메모

상태 전환
├── proposal (제안)
│   └── → negotiation (협상)
├── negotiation (협상)
│   ├── → confirmed (확정)
│   └── → proposal (되돌리기)
├── confirmed (확정)
│   └── "납품 등록" → 납품 빠른 등록 다이얼로그
└── 모든 상태에서 → canceled (취소) 가능

납품 빠른 등록 (수주에서)
├── 수주 금액 자동 채워짐
├── 납품일, 상태, 메모 입력
└── 저장 → 납품 목록에 생성됨
```

```

├── 연결 수주 선택 (선택사항 — 선택 시 고객사/금액 자동 채워짐)

├── 금액 (필수)
├── 상태 선택
├── 납품일

## 9. 전체 영업 사이클 (End-to-End)

```
【현장】 영업사원이 고객사 방문
  └── 앱에서 음성 녹음 시작
      └── 영업 내용 음성으로 기록

【STT】 음성 → 텍스트 변환 (Groq Whisper)
  └── 한국어 텍스트로 변환 → orig_memo에 자동 삽입

【AI 저장】 "AI 저장" 버튼 클릭
  ├── 영업일지 DB 저장
  └── LLM 분석 (Groq)
      ├── 고객사명 추출 → 유사 고객사 자동 매칭
      ├── 금액 정보 추출
      ├── 후속 일정 추출 → CRM_SCHEDULE에 자동 등록
      └── 요약 생성

【일정 관리】 자동 생성된 일정 확인
  ├── "제안서 제출 미팅" (예: +7일)
  └── 완료 → completed 처리

【수주 등록】 협상 진행 시 수동 등록
  ├── 제안 → 협상 → 확정
  └── 확정 시 "납품 등록" 버튼 활성화

【납품 처리】
  ├── 납품 완료 → delivered
  ├── 청구서 발송 → invoiced
  └── 수금 완료 → paid → 대시보드 매출 KPI 반영

【대시보드】
  └── 이번 달 실적 집계 자동 업데이트
```

## 11. 주요 UX 특징

1. **음성 우선 입력**: 마이크 녹음 또는 파일 첨부 → Whisper STT 자동 변환
2. **AI 자동 구조화**: LLM이 고객사 · 금액 · 일정을 텍스트에서 추출
3. **고객사 자동 매칭**: 유사 고객사 이름 제안으로 중복 방지
4. **일정 자동 생성**: AI 분석 후 후속 일정 즉시 등록
